## Writing
- Create and edit notes as needed. Never delete or rename without
  explicit instruction.
- Before creating a note, search for an existing one — extend, don't
  duplicate.
- Prefer patching a section over rewriting a whole note.

## Linking — the graph is the point
- No orphan notes: every new note links out to related notes AND gets
  linked from at least one existing note (edit those notes too).
- Link inline where the concept appears in the text, Wikipedia-style —
  not dumped in a "Related" section.
- Mention a concept that deserves its own note? Leave the [[wikilink]]
  even if the note doesn't exist yet.
- Every note should be reachable from a hub note (a topic page that
  collects and contextualizes related notes). Create hubs when a topic
  accumulates 3+ notes.

## References
- Claims that rest on an external source get the reference inline,
  next to the claim: url, or book + chapter. No reference = author's
  own thinking.

## Naming
- Titles in natural language, Title Case, singular — "Knowledge Graph",
  not "knowledge-graphs" — so wikilinks read cleanly in prose.
